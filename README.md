# LifeTrack («Пространственная ОС» для ваших целей)

**LifeTrack** — это интерактивный визуальный холст (интеллект-карта целей) с возможностью автоматического планирования. Она превращает хаотичные списки дел в **ориентированный граф**, где каждая цель, веха, задача или заметка взаимосвязаны. 

AI-ассистент работает как часть «пространственной операционной системы»: он не просто даёт текстовые ответы в чате, а **самостоятельно видит текущее состояние вашего канваса и перестраивает его на лету** (удаляет, создаёт, двигает и связывает ноды стрелками).

---

## 🌟 Возможности системы
* **Интерактивный граф (D3-zoom & Drag-and-drop):** Удобное перемещение нод по холсту, зум, создание жестких связей.
* **4 типа нод:** Цели (Goals), Вехи (Milestones), Задачи (Tasks) и Наброски/Заметки (Notes) с цветовой дифференциацией.
* **Многопользовательские пространства (Spaces):** Возможность создавать независимые проекты, каждый со своими шаблонами и графом.
* **Canvas-Aware AI Agent:** Встроенный чат-ассистент на базе **LiteLLM**. ИИ видит ваши текущие таски и может изменять граф по вашему текстовому запросу.
* **Умная декомпозиция:** Возможность в один клик отправить ноду на генеративную декомпозицию для создания связных подзадач.
* **Поддержка MCP (Model Context Protocol):** Прямая интеграция по API делает систему гибкой для сторонних агентов.

---

## 🛠️ Стек технологий

* **Frontend:** Vue 3 (Composition API / Script Setup), Vite, Tailwind CSS, TypeScript, D3.js.
* **Backend:** Python 3.12, FastAPI, Pydantic v2 (Pydantic Settings), SQLAlchemy 2 (асинхронный SQLite `aiosqlite` в разработке по умолчанию, поддержка PostgreSQL).
* **База данных:** SQLite (локальный файл `dev.db`, создается автоматически), с возможностью быстрого переключения на PostgreSQL.
* **LLM-Сервисы:** LiteLLM (автоматическая маршрутизация на GPT-4o, Anthropic, Qwen, OpenRouter, n1618 или локальную модель **Ollama**).

## 🚀 Быстрый запуск

### Шаг 1. Клонирование и подготовка
Склонируйте репозиторий:
```bash
git clone https://github.com/ваше-имя-пользователя/life-track.git
cd life-track
```

---

### Шаг 2. Запуск Backend
1. **Настройка окружения**:
   Скопируйте пример файла конфигурации:
   ```bash
   cp backend/.env.example backend/.env
   ```
   *По умолчанию в конфигурации бэкенда настроен **SQLite** (`DATABASE_URL=sqlite+aiosqlite:///./dev.db`), который автоматически развернет базу данных в файл `dev.db` при первом же запуске — настраивать Postgres не нужно!*

2. **Зависимости и окружение**:
   Создайте виртуальное окружение и установите пакеты:
   ```bash
   python -m venv backend/.venv
   source backend/.venv/bin/activate
   pip install -r backend/requirements.txt
   ```

3. **Запуск**:
   Запустите FastAPI-сервер на свободном порту `8082` (порт 8000 часто занят системными службами на macOS):
   ```bash
   uvicorn backend.app.main:app --host 127.0.0.1 --port 8082 --reload
   ```

---

### Шаг 3. Запуск Frontend
1. **Настройка окружения**:
   ```bash
   cp frontend/.env.example frontend/.env
   ```
2. **Установка зависимостей и запуск**:
   Убедитесь, что у вас установлен Node.js 18+.
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
Веб-интерфейс будет доступен по адресу: **`http://localhost:5173`**.

## 🗝️ Настройка API-ключа LLM (LiteLLM)

Ассистент умеет общаться с моделями OpenAI-стандарта. Внутри `backend/.env` выберите наиболее удобный вариант инициализации:

### Вариант 1. Кастомный OpenAI-совместимый прокси (например, n1618, OpenRouter)
```env
OPENAI_API_BASE=https://n1618.ru/api/v1          # Точка доступа прокси
OPENAI_API_KEY=n1618_your_secret_key             # Токен авторизации
LITELLM_MODEL=qwen/qwen3.5-flash-02-23          # Модель
```

### Вариант 2. Оригинальные облака OpenAI (GPT-4o)
```env
OPENAI_API_KEY=sk-proj-yourRealOpenAiKeyHere
LITELLM_MODEL=gpt-4o
```

### Вариант 3. Локально и бесплатно (через Ollama)
Скачайте клиент Ollama и запустите модель:
```bash
ollama run gemma3:2b
```
Заполните настройки:
```env
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=gemma3:2b
LITELLM_MODEL=ollama/gemma3:2b
```

## 🩹 Решение частых проблем

### Ошибка: `ERROR: [Errno 48] Address already in use`
Эта ошибка означает, что порт `8082` занят зависшим в фоне процессом.
1. Найдите PID процесса:
   ```bash
   lsof -i :8082
   ```
2. Завершите этот процесс:
   ```bash
   kill -9 <PID_из_колонки>
   ```
   *(Замените `<PID_из_колонки>` на реальный числовой PID процесса).*
